📄 OTLP Ingestion Service for MLflow-Compatible Tracing

⸻

h1. Overview

This RFC proposes a write-only OTLP ingestion service to support tracing for agentic and GenAI applications.

The service will:
	•	Accept traces via OpenTelemetry OTLP/HTTP (/v1/traces)
	•	Authenticate requests using JWT tokens
	•	Authorize access using server-side project mapping
	•	Accept MLflow experiment routing via:
	•	x-mlflow-experiment-id
	•	Normalize and convert spans into MLflow-compatible trace format
	•	Persist:
	•	Full trace JSON in S3 (source of truth)
	•	Trace metadata and spans in DynamoDB (index)
	•	Enable downstream indexing via:
	•	DynamoDB Streams → OpenSearch

The service is deployed in two regions behind a single endpoint and enforces a strict write-only access model.

h2. Architecture Overview

The system separates:
	•	Ingestion (write-only ECS service)
	•	Persistence (sync in v1, async in v2)

⸻

h2. Access Model

Producer Role

A Producer is any client sending traces.

|| Capability || Allowed ||
| Write traces | ✅ |
| Write spans/events/logs | ✅ |
| Read data | ❌ |
| Query data | ❌ |
| List data | ❌ |

This enforces a strict separation between write path and read/query systems.

h2. Multi-Region Storage and Networking

Write Model
	•	Single endpoint
	•	Single writer region per trace

⸻

DynamoDB
	•	Uses Global Tables
	•	Provides global metadata availability

Metadata fields:

{code}
writer_region
trace_json_s3_bucket
trace_json_s3_key
trace_json_s3_region
{code}

⸻

S3
	•	Region-local writes preferred
	•	Avoid cross-region dependencies
	•	Lower latency and simpler operations

⸻

Networking
	•	ECS runs inside VPC
	•	Region-local S3 access preferred
	•	No cross-region S3 dependency in v1

⸻

Read Model
	•	DynamoDB → global metadata
	•	S3 → canonical trace payload
	•	Readers must use stored S3 location metadata

h2. Example: ADK Agent

{code:python}
from opentelemetry import trace
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

JWT_TOKEN = “”
PROJECT = “project-a”
EXPERIMENT_ID = “123”

exporter = OTLPSpanExporter(
endpoint=“https://otlp.example.internal/v1/traces”,
headers={
“Authorization”: f”Bearer {JWT_TOKEN}”,
“x-project-name”: PROJECT,
“x-mlflow-experiment-id”: EXPERIMENT_ID,
},
)

provider = TracerProvider()
provider.add_span_processor(BatchSpanProcessor(exporter))
trace.set_tracer_provider(provider)

from google.adk.agents import LlmAgent
from google.adk.tools import FunctionTool

def add(a, b):
return str(a + b)

tool = FunctionTool(func=add)

agent = LlmAgent(
name=“MathAgent”,
model=“gemini-2.0-flash-exp”,
tools=[tool],
)
{code}
